# 电商用户留存与生命周期价值分析

基于 Olist 巴西电商公开数据集的用户增长、留存、复购与生命周期价值（LTV）分析项目。项目通过 `Python + Pandas + NumPy + Matplotlib + Seaborn` 对电商用户行为进行探索，重点回答“用户是否留得住、是否会复购、哪些用户更有价值”这几个核心业务问题。

## 项目亮点
- 围绕真实电商业务场景，覆盖活跃、留存、复购、流失、LTV 等关键分析主题
- 使用公开数据集完成较完整的数据清洗、字段整合与用户级分析
- 输出多类业务指标与可视化结果，适合作为数据分析作品集项目展示
- 兼顾“指标计算 + 结果解释 + 业务建议”，更贴近实际分析汇报场景

## 项目背景

随着电商行业竞争加剧，平台增长不再只依赖拉新，更依赖用户留存与长期价值提升。如果用户只完成一次购买便流失，平台将面临获客成本回收困难、GMV 增长受限、收入结构不稳定等问题。

因此，本项目从用户行为数据出发，分析平台整体活跃趋势、用户留存情况、复购特征、流失问题以及高价值用户贡献，为后续精细化运营提供依据。

## 分析目标

本项目主要回答以下问题：

1. 平台用户活跃度如何变化？
2. 用户在首购后是否持续留存？
3. 用户复购表现如何，平台是否具备较强粘性？
4. 流失用户与留存用户之间是否存在明显差异？
5. 用户生命周期价值（LTV）如何分布，高价值用户贡献有多大？

## 数据来源

- 数据集：Olist Brazilian E-Commerce Public Dataset
- 来源地址：[Kaggle - Brazilian E-Commerce](https://www.kaggle.com/datasets/olistbr/brazilian-ecommerce)

## 关键数据表

项目中主要使用了以下数据表：

- `olist_orders_dataset.csv`：订单主表
- `olist_customers_dataset.csv`：用户信息表
- `olist_order_payments_dataset.csv`：支付信息表
- `olist_order_items_dataset.csv`：订单商品表

## 核心字段

- `order_id`：订单 ID
- `customer_id` / `customer_unique_id`：用户标识
- `payment_value`：支付金额
- `order_purchase_timestamp`：订单购买时间

## 分析内容

### 1. 数据读取与预处理

- 导入订单、用户、支付、商品等多张数据表
- 合并关键字段，构建用户行为分析宽表
- 转换时间字段，便于按日、周、月进行聚合分析

### 2. 用户活跃度分析

项目计算并可视化了：

- `DAU`（日活跃用户）
- `WAU`（周活跃用户）
- `MAU`（月活跃用户）

用于观察平台整体用户规模变化和阶段性活跃波动。

### 3. 用户留存分析

- 构建用户留存矩阵
- 观察用户首购后的后续活跃表现
- 判断平台用户留存能力与用户生命周期长度

### 4. 用户复购分析

- 统计用户购买次数
- 观察用户复购率与复购分布
- 分析平台增长更多依赖新客还是老客复购

### 5. 用户流失分析

- 对比留存用户与流失用户的行为差异
- 从订单数、消费金额、客单价等维度识别潜在流失特征

### 6. 用户生命周期价值（LTV）分析

- 以用户累计消费金额作为 LTV 指标
- 观察 LTV 分布特征
- 使用消费金额分位数识别高价值用户
- 分析高价值用户对整体 GMV 的贡献

## 主要结论

结合 Notebook 中的分析结果，可以归纳出以下业务发现：

- 平台活跃用户规模整体呈增长趋势，说明用户盘子在扩大
- 用户留存与复购表现相对一般，平台对老用户的持续经营空间较大
- 用户整体消费呈现明显长尾分布，大多数用户 LTV 偏低
- 高价值用户对 GMV 贡献显著，值得重点识别和维护
- 平台后续运营重点应从“拉新”逐步转向“提留存、促复购、做分层运营”
